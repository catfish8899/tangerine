### tangerine 桌面 AI 助手 - AI 上下文说明书

#### 一、 技术栈与核心依赖
- **壳层**：Tauri 2.0 (Rust) 管理窗口及生命周期，拉起 Python Sidecar 子进程。
- **Tauri 插件**：
  - `@tauri-apps/plugin-shell`：系统 Shell 能力
  - `@tauri-apps/plugin-dialog`：原生文件选择器
  - `@tauri-apps/plugin-fs`：前端文件系统访问（当前图片预览主链路已不再依赖其完成最终显示）
- **前端**：React 18 + TypeScript + Vite 6 + Tailwind CSS v4。
- **画布**：`@xyflow/react` (React Flow 新版)。
- **渲染高亮**：`react-markdown` + `remark-gfm` + `rehype-highlight` + `highlight.js`。
- **后端图片编码依赖**：Rust 侧使用 `base64`，将本地图片转为 Data URL 返回前端显示。

#### 二、 核心架构与底层逻辑
1. **桌面双后端架构**：
   - Rust / Tauri 负责窗口、生命周期、插件注册、原生命令暴露、Python Sidecar 拉起。
   - Python FastAPI 负责 `/chat` 推理流式接口与模型适配。
2. **SSE 流式传输**：FastAPI `/chat` 返回 `StreamingResponse`。前端 fetch 解析器内置 `extractTextDelta` 管道，支持多格式混吐。
3. **角色系统**：唯一性重名校验；仅在空白会话可选；提示词通过 API `system` 通道发送，不污染展示气泡。
4. **Ollama 高抗错**：直连本地 `11434` 端口 `/v1` 兼容网关（OpenAI SDK），抹平 SSE 缓冲差异。
5. **流式熔断防御**：握手异常时触发 `Try-Catch Stream Breaker`，“抹杀”空临时 AI 消息，追加 `system_err` 气泡。
6. **UI 加载状态**：字出即隐藏“思考中”；流式结束后淡入 Metadata（提供商、Token 消耗、耗时）。
7. **工作流画布隔离**：`ChatSession.type` 区分 `"chat"` 与 `"automation"`。数据按 `tangerine_automation_flow_{sessionId}` 隔离存储于 LocalStorage，随会话销毁物理联动删除。
8. **附件接入方式**：
   - 支持通过 **Tauri 原生文件选择器** 选择文件；
   - 支持通过 **Tauri 窗口原生拖放事件** 拖入文件；
   - 附件数据以绝对路径形式参与后续会话发送。
9. **图片预览最终稳定方案**：
   - 前端不再依赖 `convertFileSrc` 直接映射本地路径；
   - 也不再将图片预览完全依赖前端 `fs.readFile + blob URL`；
   - 当前稳定实现为：**前端调用 Rust 命令 `read_image_as_data_url`，Rust 直接读取本地图片文件并编码为 `data:image/...;base64,...` 返回前端**；
   - 该方案用于解决 Tauri WebView 下本地图片灰块、裂图、scope/权限链路不稳定等问题。
10. **图片预览故障定位思路（已验证）**：
   - 先前问题不在图片类型识别；
   - 主要失败点发生在 WebView 对本地路径资源加载、或前端 fs 读取授权/作用域链路；
   - 改为 Rust 直读后，图片缩略图与预览恢复正常。

#### 三、 文件与图片处理逻辑（新增补充）
1. **文件选择**：
   - 通过 `@tauri-apps/plugin-dialog` 的 `open()` 打开系统原生选择器；
   - 解决浏览器风格文件框在桌面环境中的兼容和显示问题。
2. **文件拖拽**：
   - 通过 `@tauri-apps/api/window` 的 `onDragDropEvent` 监听窗口级拖放；
   - 避免依赖浏览器 `dataTransfer.files` 在桌面端路径不稳定的问题。
3. **附件识别**：
   - 前端通过 `getFileType(fileName)` 将附件分为 `image / audio / code / office / other`。
4. **图片预览生成**：
   - 图片文件会在进入附件列表时调用 Rust 命令生成 `previewUrl`；
   - `previewUrl` 当前为 Data URL，可直接赋值给 `<img src>`；
   - 若失败则写入 `previewError`，供前端阻断预览并提示。
5. **大图预览**：
   - 点击已成功生成缩略图的图片附件，可进入大图预览层；
   - 若 `previewUrl` 缺失，则不会进入预览，而是走失败提示逻辑。
6. **发送阶段**：
   - 聊天请求发送给 Python Sidecar 时，仍携带原始绝对路径 `file_paths`；
   - 图片缩略图显示与模型侧文件处理是两条链路，互不冲突。

#### 四、 画布核心组件与交互（Pointer 模拟拖拽）
- **拖拽机制**：因 Tauri WebView 兼容性，放弃原生 HTML5 拖拽，全量采用 **Pointer 事件模拟**（`onPointerDown` -> `window pointermove` -> `window pointerup` -> `screenToFlowPosition` -> `setNodes`）。
- **类目化生成**：底部工具栏仅有“角色/工具/开关/集合/对话/硬件/定时”七大类目按钮。拖入画布后，在节点**内部下拉选单**选择具体细分项。
- **节点特征**：
  * `RoleNode`：下拉选单聚焦/触发时即时读取 LocalStorage `tangerine_roles`。兼容 `systemPrompt`/`prompt`/`description`。
  * `ToolNode`：内置 `Tavily / Web Search`。
  * `TriggerNode`：内置“开始对话”。
  * `CollectionNode`：集合区域，使用 `NodeResizer` 调整大小。
  * `ConversationNode`：内置“文本对话”，带 `.nodrag` 多行 `textarea`。
  * `HardwareNode`：内置“连接硬件”。
  * `TimerNode`：结构化下拉选择（日期、上午/下午、时、分、秒），最终持久化字符串格式如：`2026年06月27日 上午 9:17:37`。
- **快捷键与历史机制**：
  * **焦点防御**：监听头部执行 `isEditableElement(event.target)` 拦截 input/textarea/select，防止编辑时误触发快捷键。
  * **撤销重做 (Ctrl+Z)**：纯函数 `cloneFlowSnapshot` 深度拷贝状态，最大深度 80 步。
  * **彻底删除 (Delete/Backspace)**：级联删除集合节点及其内部吸附的子孙节点与连线。
  * **复制粘贴 (Ctrl+C / Ctrl+V)**：利用 `getDescendantNodeIds` 深度复制集合及其内部所有子节点和关联 edges，粘贴时以鼠标流坐标为中心计算 delta offset 偏移量，并赋予全新唯一抗冲突 ID。

#### 五、 已确认的桌面端开发结论（新增）
1. Windows 11 下当前问题场景**不是手机式系统弹窗授权模型**。
2. Tauri 前端本地文件访问的关键限制点通常在：
   - capability
   - fs permission
   - fs scope
   - WebView 本地资源映射方式
3. 对于聊天输入区图片缩略图这类高频预览功能，**Rust 直读文件并返回 Data URL** 是当前项目中更稳、更易维护、更便于定位问题的方案。
4. Rust 终端输出比前端控制台更适合作为 Tauri 本地文件读取链路的调试依据。
