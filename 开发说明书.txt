### tangerine 桌面 AI 助手 - AI 开发说明书

#### 一、项目定位
- 桌面端 AI 助手，包含：
  - 普通聊天会话
  - 角色系统（System Prompt）
  - 自动化工作流画布
  - 本地 / 云端模型接入
- 本文档仅用于 **AI 辅助开发快速指路**，重点提供：
  - 技术栈
  - 核心架构
  - 关键约定
  - 文件树定位

---

#### 二、技术栈与核心依赖
- **桌面壳层**：Tauri 2（Rust）
- **前端**：React 18 + TypeScript + Vite 6 + Tailwind CSS v4
- **工作流画布**：`@xyflow/react`
- **Markdown 渲染**：
  - `react-markdown`
  - `remark-gfm`
  - `rehype-highlight`
  - `highlight.js`
- **Tauri 插件**：
  - `@tauri-apps/plugin-shell`
  - `@tauri-apps/plugin-dialog`
  - `@tauri-apps/plugin-fs`
- **Python Sidecar**：FastAPI
- **Rust 图片处理**：`base64`

---

#### 三、核心架构
1. **双后端桌面架构**
   - Rust / Tauri：窗口、生命周期、插件注册、原生命令、拉起 Python Sidecar
   - Python / FastAPI：`/chat` 推理接口、SSE 流式输出、模型适配

2. **聊天主链路**
   - 前端通过 fetch + 流式解析消费 SSE
   - 文本增量通过统一提取管道进入消息气泡
   - system prompt 通过 API 的 `system` 通道传入，不直接污染聊天展示气泡

3. **模型接入**
   - 支持云端模型与本地 Ollama
   - Ollama 走本地 `11434/v1` OpenAI 兼容接口
   - 设置页负责模型提供商、模型列表、保存与同步

4. **角色系统**
   - 角色数据存储于 LocalStorage：`tangerine_roles`
   - 角色字段核心为：
     - `id`
     - `name`
     - `systemPrompt`
     - `provider`
     - `model`
   - 角色名要求唯一，保存时做重名校验
   - 可绑定默认提供商和模型，也可不绑定

5. **自动化工作流**
   - 会话类型通过 `ChatSession.type` 区分：
     - `"chat"`
     - `"automation"`
   - 自动化画布按会话 ID 隔离存储：
     - `tangerine_automation_flow_{sessionId}`

6. **附件与图片预览**
   - 文件选择使用 Tauri 原生文件选择器
   - 文件拖入使用 Tauri 窗口级拖放事件
   - 聊天发送时传递原始绝对路径 `file_paths`
   - 图片预览不走前端本地路径直显，改为：
     - 前端调用 Rust 命令 `read_image_as_data_url`
     - Rust 读取图片并返回 Data URL
   - 该方案是当前桌面端稳定方案

---

#### 四、关键交互与开发约定
1. **消息显示**
   - AI 输出为流式追加
   - 一旦收到有效文本增量，即隐藏“思考中”
   - 流式结束后再补充显示元数据（提供商、Token、耗时等）
   - 流式异常时，移除空 AI 临时消息，改为追加 `system_err` 气泡

2. **角色系统 UI 约定**
   - `RolesModal.tsx` 负责角色增删改查
   - 保存角色按钮交互：
     - 默认文案：`保存角色`
     - 不显示默认勾图标
     - 点击保存成功后按钮变绿
     - 成功态文案：`已保存`
     - 若继续编辑内容，则恢复默认保存态
   - 该交互应与“保存模型”区域保持一致风格

3. **工作流画布交互**
   - 因 Tauri WebView 兼容性，放弃原生 HTML5 拖拽
   - 全量采用 Pointer 事件模拟拖拽
   - 支持：
     - 撤销 / 重做
     - 复制 / 粘贴
     - 集合节点及其子孙级联删除
   - 编辑输入元素时需拦截全局快捷键误触发

4. **文件处理约定**
   - 预览链路与模型处理链路分离
   - 图片缩略图仅依赖 `previewUrl`
   - 若后续扩展音频、Office、更多本地预览能力，优先沿用：
     - Rust 原生读取
     - 前端展示

---

#### 五、数据存储约定
- 角色列表：`tangerine_roles`
- 模型设置：`tangerine_api_settings`
- 自动化流程：`tangerine_automation_flow_{sessionId}`

前端事件广播：
- 角色更新后派发：`tangerine_roles_changed`

七、当前开发结论
Python Sidecar 自动拉起正常
Tauri 主窗口启动正常
会话、角色、自动化工作流均已具备本地持久化
文件选择与拖放已接入桌面原生能力
图片缩略图与大图预览已采用 Rust Data URL 方案并恢复稳定
后续新增桌面本地预览能力时，优先考虑“Rust 读取 + 前端展示”的稳定链路

八、AI 辅助开发建议
修改脚本时优先保持现有业务逻辑不变
先从以下文件定位需求：
聊天主逻辑：src/hooks/useChatManager.ts
输入区与附件：src/components/ChatInput.tsx
单条消息展示：src/components/MessageItem.tsx
模型设置：src/components/SettingsModal.tsx
角色系统：src/components/RolesModal.tsx
自动化主编辑器：src/components/automation/FlowEditor.tsx
Rust 原生命令：src-tauri/src/lib.rs
Python 推理接口：src-tauri/py_sidecar/main.py
若涉及桌面文件访问、图片预览、系统能力调用，优先检查：
src-tauri/src/lib.rs
src-tauri/capabilities/default.json
src-tauri/tauri.conf.json
