### tangerine 桌面 AI 助手 - AI 上下文说明书

#### 一、 技术栈与核心依赖
- **壳层**：Tauri 2.0 (Rust) 管理窗口及生命周期，拉起 Python Sidecar 子进程。使用 `@tauri-apps/plugin-shell`。
- **前端**：React 18 + TypeScript + Vite 6 + Tailwind CSS v4。
- **画布**：`@xyflow/react` (React Flow 新版)。
- **渲染高亮**：`react-markdown` + `remark-gfm` + `rehype-highlight` + `highlight.js`。

#### 二、 核心架构与底层逻辑
1. **SSE 流式传输**：FastAPI `/chat` 返回 `StreamingResponse`。前端 fetch 解析器内置 `extractTextDelta` 管道，支持多格式混吐。
2. **角色系统**：唯一性重名校验；仅在空白会话可选；提示词通过 API `system` 通道发送，不污染展示气泡。
3. **Ollama 高抗错**：直连本地 `11434` 端口 `/v1` 兼容网关（OpenAI SDK），抹平 SSE 缓冲差异。
4. **流式熔断防御**：握手异常时触发 `Try-Catch Stream Breaker`，“抹杀”空临时 AI 消息，追加 `system_err` 气泡。
5. **UI 加载状态**：字出即隐藏“思考中”；流式结束后淡入 Metadata（提供商、Token 消耗、耗时）。
6. **工作流画布隔离**：`ChatSession.type` 区分 `"chat"` 与 `"automation"`。数据按 `tangerine_automation_flow_{sessionId}` 隔离存储于 LocalStorage，随会话销毁物理联动删除。

#### 三、 画布核心组件与交互（Pointer 模拟拖拽）
- **拖拽机制**：因 Tauri WebView 兼容性，放弃原生 HTML5 拖拽，全量采用 **Pointer 事件模拟**（`onPointerDown` -> `window pointermove` -> `window pointerup` -> `screenToFlowPosition` -> `setNodes`）。
- **类目化生成**：底部工具栏仅有“角色/工具/开关/集合/对话/硬件/定时”七大类目按钮。拖入画布后，在节点**内部下拉选单**选择具体细分项。
- **节点特征**：
  * `RoleNode`：下拉选单聚焦/触发时即时读取 LocalStorage `tangerine_roles`。兼容 `systemPrompt`/`prompt`/`description`。
  * `ToolNode`：内置 `Tavily / Web Search`。
  * `TriggerNode`：内置“开始对话”。
  * `CollectionNode`：集合区域，使用 `NodeResizer` 调整大小。
  * `ConversationNode`：内置“文本对话”，带 `.nodrag` 多行 `textarea`。
  * `HardwareNode`：内置“连接硬件”。
  * `TimerNode`：结构化下拉选择（日期、上午/下午、时、分、秒），最终持久化字符串格式如：`2026年06月27日 上午 9:17:37`。
- **快捷键与历史机制**：
  * **焦点防御**：监听头部执行 `isEditableElement(event.target)` 拦截 input/textarea/select，防止编辑时误触发快捷键。
  * **撤销重做 (Ctrl+Z)**：纯函数 `cloneFlowSnapshot` 深度拷贝状态，最大深度 80 步。
  * **彻底删除 (Delete/Backspace)**：级联删除集合节点及其内部吸附的子孙节点与连线。
  * **复制粘贴 (Ctrl+C / Ctrl+V)**：利用 `getDescendantNodeIds` 深度复制集合及其内部所有子节点和关联 edges，粘贴时以鼠标流坐标为中心计算 delta offset 偏移量，并赋予全新唯一抗冲突 ID。
