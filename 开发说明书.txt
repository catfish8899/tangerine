# tangerine 桌面 agent框架 - 开发说明书

## 一、技术栈与核心依赖

- **桌面壳层**：Tauri 2（Rust）
- **前端**：React 18 + TypeScript + Vite 6 + Tailwind CSS v4
- **工作流画布**：@xyflow/react
- **Markdown 渲染**：
  - react-markdown
  - remark-gfm
  - rehype-highlight
  - highlight.js
- **Tauri 插件**：
  - @tauri-apps/plugin-shell
  - @tauri-apps/plugin-dialog
  - @tauri-apps/plugin-fs
- **Python Sidecar**：FastAPI
- **Rust 图片处理**：base64

## 三、架构设计与解耦说明

### 1. 聊天核心状态解耦 (`useChatManager`)
为避免单一 Hook 文件过于臃肿导致维护困难和性能瓶颈，`useChatManager` 采用了**组合式自定义 Hook** 的设计模式，按职责拆分为 5 个子模块：
- **`useChatUIState`**：纯粹管理 UI 交互状态（如输入框文本、菜单显隐、拖拽状态），隔离 DOM 事件监听。
- **`useModelManager`**：专注处理 API 提供商配置解析、模型列表同步以及基于角色的模型推断。
- **`useAttachmentManager`**：封装 Tauri 原生文件选择器、拖放事件监听以及 Rust 侧图片 Base64 预览生成。
- **`useSessionManager`**：管理 LocalStorage 持久化、会话树操作（包含复杂的分支切换与深度删除防复活逻辑）。
- **`useStreamChat`**：处理与 Python Sidecar 的 SSE 流式通信、数据解析、中断控制（AbortController）及错误降级。

**优势**：主入口 `useChatManager.ts` 仅负责依赖注入和跨模块副作用联动，对外暴露的 `state`、`refs` 和 `actions` 接口保持 100% 兼容，调用方（如 `App.tsx`）无需感知内部重构。

### 2. 多模型与算力调度
- **AMD 生态兼容**：框架不依赖任何 NVIDIA 专属生态（如 CUDA 强绑定），Ollama 侧车支持 AMD ROCm 或纯 CPU 推理，云端模型通过标准 OpenAI 兼容 API 调用。
- **算力保护**：Python Sidecar (`main.py`) 通过 FastAPI 的 `Request.is_disconnected()` 实时检测前端 SSE 连接状态。若用户点击“停止生成”或关闭应用，Sidecar 会立即切断推理算力，避免资源浪费。
