### tangerine 桌面 AI 助手 - 阶段性开发说明书

本项目是一款基于 Windows 11 Fluent Design（暗黑微光风格）的 AI 智能聊天客户端。应用后端使用 Rust (Tauri 2.0) 管理窗口及生命周期，并通过本地 Python Sidecar 服务桥接调用云端 DeepSeek API 及本地 Ollama 服务，确保 API 密钥的安全性和本地扩展能力。

--------------------------------------------------------------------------------------------------------------------------------

#### 一、 技术栈组成 & 新增前端依赖
**宿主与壳层 (Desktop Shell)**
- Tauri 2.0：轻量级桌面开发框架。使用 Rust 构建，负责管理应用窗口、文件系统访问，以及在后台自动拉起和管理 Python Sidecar 子进程。

**前端界面 (Frontend UI)**
- React 18 & TypeScript：用于高效率的前端组件开发与强类型保证。
- Vite 6：极其快速的现代化前端构建与热重载工具。
- Tailwind CSS v4 (@tailwindcss/postcss)：采用全新 v4 架构。通过引入 `@import "tailwindcss";` 结合 `@source` 动态规则，极速提取 Fluent 风格组件样式。
- Lucide React：现代、简洁、一致的图标库。

**前端渲染与高亮依赖**
为了实现高可读性的 AI 消息回复、代码框样式与高亮：
- react-markdown: 用于解析并渲染 AI 返回的 Markdown 数据。
- remark-gfm: 插件，支持 GitHub 风格的 Markdown（表格、任务列表等）。
- rehype-highlight & highlight.js: 插件及主题包，用于解析代码块并提供 Atom One Dark 彩色高亮样式。
安装命令：
  `pnpm add react-markdown remark-gfm rehype-highlight highlight.js`

#### 三、 项目环境依赖与多电脑协同开发
为了能在不同的电脑上通过 GitHub 克隆代码后快速恢复开发，请确保目标电脑满足以下环境：

**1. 全局系统环境要求**
- Node.js：建议安装 v18.x 或 v20.x LTS 版本。
- pnpm：推荐使用的前端包管理器。
- Rust 编译链：安装 Rustup，且 Windows 电脑需要 C++ 生成工具（VS C++ 桌面开发工作负载）。
- Python 3：确保 python 已添加到系统环境变量 PATH 中。
- 环境变量：系统中建议保留 `DEEPSEEK_API_KEY` 作为云端首选秘钥（后续可通过设置面板自定义配置）。
- 本地服务（可选）：若要启用离线 AI，建议安装本地 Ollama 并拉取相应模型（如 `qwen2.5` 等）。

**2. 跨电脑克隆后的快速启动步骤**
在新电脑上克隆仓库后，只需在项目根目录下依次执行以下命令：

- **步骤 A**：安装前端依赖
  `pnpm install`

- **步骤 B**：安装 Python Sidecar 依赖
  `pip install -r src-tauri/py_sidecar/requirements.txt`

- **步骤 C**：清理缓存并一键启动开发预览
  如遇到样式未能实时加载等现象，建议清理 Vite 缓存：
  PowerShell: `Remove-Item -Recurse -Force .vite`
  启动开发服务：`pnpm tauri dev`

#### 四、 核心逻辑
**LocalStorage 多轨持久化 & 动态事件通信**：
- 聊天历史：持久化在 LocalStorage 的 `"tangerine_chat_sessions"` 中。
- 多 API 配置（混合配置）：持久化在 LocalStorage 的 `"tangerine_api_settings"` 中。在本地模型设置页面运行自动检索时，Ollama 模型的元数据会作为一个独立的提供商模块存入并更新该配置。
- 动态配置更新事件：设置页面更新 API 后，通过分发自定义事件 `tangerine_api_settings_changed`，可被主页面 App 动态监听捕获，从而实现主页面右下角可用模型下拉列表的**无刷新实时刷新与同步**。
- 对话框字号：持久化在 LocalStorage 的 `"tangerine_font_size"` 中（默认 12px，跨组件订阅 `"storage_font_size_changed"` 事件实时同步更新）。

**本地与云端配置完全解耦隔离**：
- 云端模型设置（原“模型 API 设置”）过滤掉 `"ollama"`，仅渲染和更改外部大语言模型。
- 本地模型设置不仅可以进行本地 Ollama 的心跳和模型拉取，而且在其底部整合了专属于 Ollama 路径与可调用模型名管理的高级面板。通过该架构隔离，杜绝了导入本地模型时顶替、抢占原有云端 API 配置行的配置冲突 Bug。

**网络流智能分流**：
- 在调用统一请求路由 `/chat` 时，Python 后端会根据客户端请求的 `model` 名字进行检索：若包含 `"deepseek"` 特征字则走向官方 OpenAI 客户端通道；否则走向本地 Ollama 的 `v1/chat/completions` 通道进行非流式推理。

**气泡级右键删除机制**：
- 支持右键点击聊天气泡删除单条信息，删除后该消息将在会话数组中物理移除，确保在后续新发送请求时，完全不计入发送给 API 的上下文。

**API 异常防御与对话守卫**：
- 对话交替校验：发送新问题或在旧消息节点启用分支发送时，系统会严格过滤系统报错并核对剩余消息顺序。如果不满足 `[User, AI, User, AI]` 交替对应的逻辑结构，发送动作会被拦截，并弹出一个 3 秒后自动淡出的警示浮窗。
- Token 深度捕获：升级了多路径冗余捕获算法，彻底兼容各大云端提供商及本地 Fast API 转发中 `usage.total_tokens`、`usage_metadata`、`total_tokens` 等各种格式的 Token 字段解析，保障元数据完整呈现。
