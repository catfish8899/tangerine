# tangerine 桌面 agent框架 - 开发说明书（新）

## 一、技术栈与核心依赖

- **桌面壳层**：Tauri 2（Rust）
- **前端**：React 18 + TypeScript + Vite 6 + Tailwind CSS v4
- **工作流画布**：@xyflow/react
- **Markdown 渲染**：
  - react-markdown
  - remark-gfm
  - rehype-highlight
  - highlight.js
- **Tauri 核心与插件**：
  - `@tauri-apps/api` (核心 API、路径处理)
  - `@tauri-apps/plugin-shell`
  - `@tauri-apps/plugin-dialog`
  - `@tauri-apps/plugin-fs` (本地文件系统读写)
- **Python Sidecar**：FastAPI
- **Rust 图片处理**：base64 (用于读取本地图片生成 Data URL 预览)
- **本地持久化方案**：JSON 文件存储 + 多模态文件物理原样存储 (替代原 LocalStorage)

## 三、存储架构设计与重构进度 (恢复点)

### 1. 存储架构概述
框架已彻底废弃 `localStorage`，全面迁移至 **JSON 文件 + 多模态文件物理原样存储** 架构：
- **结构化数据**（会话、消息、画布节点、设置）：序列化为 JSON 文件，存储在 `$APPLOCALDATA` 下的分类目录中（如 `sessions/`, `canvas/`, `settings/`）。
- **多模态文件**（图片、文档）：用户选择或拖拽的文件会被**物理复制**到 `$APPLOCALDATA/attachments/{sessionId}/` 目录下。JSON 中仅存储相对路径（`localRelativePath`）。
- **前端渲染**：通过 Rust 侧的 `read_image_as_data_url` 命令读取本地复制后的图片，生成 Base64 Data URL 供 `<img>` 渲染，彻底规避 Tauri v2 Asset 协议的跨域与 Scope 坑。
- **后端通信**：发送消息时，前端将相对路径异步转换为本地绝对路径，传递给 Python Sidecar 直接读取，避免 Base64 传输带来的内存与带宽开销。

  - **目标**：重构 `src/components/automation/automationStorage.ts` 和 `useFlowPersistence.ts`。

- **第五步：设置与角色管理持久化重构**
